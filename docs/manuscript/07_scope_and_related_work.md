# Scope and related work {#sec:scope}

NoOrg draws on several established ideas without claiming to reproduce their full theories. Agent-oriented programming provides a vocabulary for autonomous software entities and coordination [@wooldridge-jennings-1995]. The actor model treats isolated entities and message passing as a basis for concurrent computation [@hewitt-bishop-steiger-1973]. Logical clocks and event ordering explain why identifiers and delivery records matter when asynchronous operations are observed [@lamport-1978]. Durable state envelopes and atomic replacement apply well-known persistence engineering principles to a single-process file store.

The system does not implement distributed consensus. The classic impossibility result for deterministic consensus in an asynchronous system with one fault [@fischer-lynch-paterson-1985] is included as a boundary marker: NoOrg’s single-writer file lock is a process-local coordination mechanism, not a proof of distributed agreement. Similarly, schema validation constrains representation and does not establish semantic truth. Reproducible computational research practices motivate the separation of source, generated output, provenance, and evidence [@sandve-2013; @wilson-2014].

The scope is therefore precise. NoOrg offers a typed task lifecycle, explicit provider boundary, local persistence, event delivery, health signals, and a release manuscript. It does not offer cross-host scheduling, consensus, exactly-once effects in external systems, broad provider benchmarking, or an empirical account of agent cognition.
