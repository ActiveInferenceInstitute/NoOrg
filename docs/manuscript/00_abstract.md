# Abstract {#sec:abstract}

NoOrg is a TypeScript runtime for coordinating explicitly registered agents through a durable task state machine. This manuscript reconstructs the system from a small set of observable requirements: a task may be marked complete only after a registered agent executes it and its result satisfies a declared schema; state transitions must be persisted atomically; external-model access must be selected by configuration; and lifecycle resources must be drained on shutdown. The implementation uses constructor-injected dependencies, a versioned file state envelope, typed event delivery, bounded task execution, and two explicit provider profiles. The local profile derives results deterministically from domain functions; the OpenAI profile uses structured responses and validates them before completion.

The contribution is an evidence-bound runtime design and release process, not a claim about general model intelligence or system-scale performance. The evaluation records the repository’s executable checks, a real coordinator execution, persisted state recovery, provider metadata, event delivery, HTTP readiness, and artifact checksums. Values in the rendered manuscript are hydrated from that evidence at release time: {{EVIDENCE_TEST_COUNT}} tests passed across the local validation run, {{EVIDENCE_COMPLETED_STATUS}} was observed for the coordinator task, and the corpus validator inspected {{EVIDENCE_UNIT_FILES}} Markdown files.
